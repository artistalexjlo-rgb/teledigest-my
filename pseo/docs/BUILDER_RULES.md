# Luky pSEO-билдер — УСТАНОВЛЕННЫЕ правила (не переоткрывать)

> Это зафиксированные выводы, выстраданные в обсуждении. НЕ переспрашивать, НЕ
> «улучшать откатом», НЕ начинать с нуля. Меняем только осознанно и явно.
> Источник тона — рядом `MESSAGING.md`. Это — про ЧТО публиковать и КАК не соврать.

---

## 1. Grounded-only. Ни одного выдуманного факта.

Экстракт ≠ генерация. Клод на странице ТОЛЬКО сжимает/переписывает реально найденные
ответы из `wisdom_base_local`. Ничего не дописывает «от себя». Промпт обязателен:
«только то, что есть в источниках; нет источника — нет факта».

⚠️ Образец `ru/br/finance` был МОКАП дизайна — факты в нём авторские, НЕ из данных.
Мокапу так можно. Билдеру — НЕЛЬЗЯ. Галлюцинация правдоподобного факта хуже мух:
это уверенная дезинформация на публичном SEO, прямо против оффера «без воды, из живого опыта».

## 2. Провенанс обязателен.

Каждый факт тянется к исходным сообщениям + `source_count`. Уровень доверия факта:
- **≥2 независимых источника** → подаём как факт.
- **1 источник** → анекдот одного человека: либо выкинуть, либо явно смягчить
  («по одному отзыву…»), НЕ в short_answer, НЕ как утверждение. (waze: multi-source confirmation.)

## 3. Что публиковать — две оси, НЕ одна частотность.

Раньше ошибочно свёл к «частое=страница / редкое=ассистент». НЕВЕРНО. Решает пересечение:

|                | **Данные есть**                                   | **Данных нет**                        |
|----------------|---------------------------------------------------|---------------------------------------|
| **Частый запрос** | страница, но БОЙ с топами (у всех есть статьи)  | дыра в грабе — тема болит, не добрали |
| **Редкий запрос** | 🔥 ЗОЛОТО: низкая конкуренция × твои уникальные данные | → ассистент (или шум)            |

**🔥 Ров — в лонг-тейле, не в частом.** Где совпало: топы не воюют И только в ТВОИХ
чатах есть ответ («спот для удалёнки с генератором на пляже X»). Generic-визу делает
каждый — это НЕ ров. Специфичный интент + grounded-ответ, которого нет у топов — ров.
Конкурент это руками не соберёт.

- редкое-БЕЗ-данных → ассистент (живой ретривал, ответ под случай).
- редкое-С-данными → самая ценная страница. Хвост распределения = месторождение, не балласт.
- Приоритет билдера: вскрывать смачные пары (узкий интент × grounded), НЕ гнать generic.

## 4. Глубина страницы = ширина запроса.

«Тонко» — это НЕ «мало блоков», а «нечем подтвердить». Узкому точному лонг-тейл-запросу
хватит ОДНОГО тугого ответа с 2-3 источниками — это норм страница. Не раздувать ради объёма.
Широкой теме (виза/деньги) — да, нужна плотность (≥5-6 grounded Q&A + short_answer).

## 5. Тонкий-БЕЗ-данных не выходит.

Если по теме нечем подтвердить — страница не публикуется (тонкие pSEO = яд для домена).
Ждём пока награбится; sweeper догонит, тема перейдёт границу. Обратимо.

## 6. Воронка зашита в каждую страницу.

CTA «твоя ситуация особая? спроси Luky». Страница = общая/находимая боль (бесплатный вход),
хвост (личное/редкое/без-данных) перетекает в продукт-ассистент. Длинный хвост не
выбрасывается — он причина открыть Luky.

## Стартовый шаг (когда садимся за билдер)

НЕ «выбрать порог числом». Снять с живого `wisdom_base_local` картину geo×topic:
сколько уникальных Q и какой source_count. Это даёт: (а) очередь — что плотное уже сейчас;
(б) смачные лонг-тейл-пары (узкий интент × grounded, низкая конкуренция); (в) дыры
(частая боль без данных = граб недобирает). Порог ставим по факту, обратимо — не по вайбам.

Первый реальный тест: заменить фикстуру `wisdom_br_sample.json` живым запросом к
`wisdom_base_local` по BR/finance → прогнать grounded-экстракт → глазами сверить с мокапом.
Держит планку — масштабируем. Жиже — допиливаем экстракт-промпт. НЕ дописывать факты, чтоб «дотянуть».
